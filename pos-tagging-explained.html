01

What POS tagging actually is

Part-of-speech (POS) tagging is the process of parsing a piece of text and labeling every single word with its grammatical role — noun, verb, adjective, pronoun, and so on. In effect, it's an automated grammatical breakdown ("i'raab") of a sentence.

Once every word has a role attached to it, an application can understand the implicit meaning of a sentence rather than just the raw characters. That single capability — knowing what each word is doing — turns out to be the foundation for a surprising number of real products.

02

Why it matters: real applications

Conversational AI

Chatbots like ChatGPT

A chatbot can't reply to a question about the present with an answer set in the past, or vice versa. By tagging each word's role and tense, the model detects which timeframe you're speaking in and responds in that same timeframe — unless you explicitly ask it to predict the future.

Search

Information retrieval

When you search Google (or any search engine), it analyzes the implicit meaning of your query — which words are the key nouns, actions, etc. — to return the most relevant results instead of just matching characters.

Translation

Machine translation

A word can be a noun in one context and a verb in another. Without knowing a word's actual grammatical role in the sentence, a translator could mistranslate a noun as a verb or a verb as a noun.

Classification

Sentiment analysis

When classifying text as positive or negative, the part-of-speech of each word (especially adjectives and verbs) helps the model understand the intended sentiment behind the sentence.

Preprocessing

Text filtering

POS tags let you strip out things you don't want — punctuation, whitespace tokens, unrecognized ("junk") tokens — before further processing, which is demonstrated later in this lesson.

03

Doing it with spaCy

This course has used NLTK exclusively so far. This lesson introduces a second library, spaCy, and shows how it handles POS tagging.

Setup

Run once in your notebook (skip if already installed):
# Install spaCy !pip install spacy # Download the pretrained English model !python -m spacy download en_core_web_sm

Loading the model and tagging a sentence

import spacy nlp = spacy.load("en_core_web_sm") # the pretrained pipeline doc = nlp("your text goes here") for token in doc: print(token.text, token.pos_, token.explain(or spacy.explain(token.pos_)))

Every token comes back with a short label (PRON, VERB, ADP, NOUN…). These abbreviations aren't self-explanatory at first glance — spaCy's own documentation site lists every tag with a plain description, or you can call spacy.explain() on any tag to get its full name back, e.g. passing "PRON" returns "pronoun".

04

Coarse tag vs. fine-grained tag

token.pos_ gives you the coarse grammatical category (noun, verb, adjective…). But spaCy also exposes token.tag_, which is far more specific — for example, it distinguishes past tense verbs from present tense verbs, not just "verb" in general.

for token in doc: print(token.text, token.tag_, spacy.explain(token.tag_))

Example — tense flips with one letter

Given "He quits the shop", spaCy correctly tags quits as present tense. Change nothing but the ending — "He quit the shop" — and spaCy correctly flips the tag to past tense. It's reading the word's form and its context in the sentence, not just matching it to a fixed dictionary entry.

Takeaway — spaCy isn't just memorizing "this word = this tag." It's genuinely parsing the sentence structure to decide a word's role and tense based on context, which is exactly what makes tense-aware chatbot replies possible.
05

Using POS tags to filter text

A very practical use of POS tagging is cleaning up a document: stripping whitespace tokens, punctuation, and "junk"/unrecognized tokens (spaCy marks these with the tag X), so only meaningful words remain.

filtered_tokens = [] for token in doc: if token.pos_ not in ["SPACE", "PUNCT", "X"]: filtered_tokens.append(token) print(filtered_tokens[:20])

Running this over a messy document (one with stray symbols, extra spaces, and unrecognized abbreviations like "etc") removes every space token, every punctuation mark (exclamation points, colons, semicolons, commas, periods…), and every unrecognized token — leaving a clean list of real words.

06

Counting POS categories across a document

If you want to know how many nouns, verbs, or adjectives appear in a whole document, spaCy's doc.count_by() function (paired with spaCy's attribute constant spacy.attrs.POS) returns a frequency count keyed by an internal numeric ID for each tag.

counts = doc.count_by(spacy.attrs.POS) print(counts) # -> {96: 13, 92: 46, ...} (numeric ID: frequency)

Those numeric IDs aren't readable on their own, so you look each one up in the document's vocabulary to get back its actual tag name:

print(doc.vocab[96].text) # -> "PRON"

To turn the whole frequency table into something readable in one pass:

for k, v in counts.items(): print(doc.vocab[k].text, v) # -> PRON 13 # -> NOUN 46 # -> ...

This is useful for corpus-level analysis — e.g. reporting that a body of text contains 13 verbs, 46 nouns, 24 adverbs, and so on.

07

The same idea in NLTK

spaCy isn't the only way to do this — NLTK, the library used throughout the rest of the course, has its own POS tagger and it's just as simple to call.

import nltk tokens = nltk.word_tokenize(text) # split sentence into words tags = nltk.pos_tag(tokens) # tag each word for word, tag in tags: print(word, tag)

NLTK's tag abbreviations (DT, JJ, VBZ…) follow the same idea as spaCy's — for example, "quick" comes back tagged JJ, which the Penn Treebank tag reference describes as adjective. If a tag ever looks unfamiliar, the same strategy applies: look it up in the tag-set reference to see its plain-English meaning.

08

Wrap-up

POS tagging turns out to be a small, mechanical-sounding step — label every word with its grammatical role — that quietly underlies chatbots that keep their tense straight, search engines that understand intent, translators that don't confuse nouns and verbs, sentiment classifiers, and basic text cleaning. Both spaCy (pos_ / tag_) and NLTK (pos_tag) make it a two- or three-line operation, and both were demonstrated tagging the same kinds of sentences to produce comparable results.
