Introduction

The Big Picture: Why Preprocessing Matters

In any AI pipeline, raw data is never fed straight into a model — it first goes through preprocessing. In Computer Vision, this means taking images and applying filters, smoothing, denoising, or domain transformations before the pixels ever reach the model. Once the images are cleaned up, they're converted into arrays of numbers that a machine learning or deep learning model can actually learn from.

Natural Language Processing follows the exact same logic, just with text instead of pixels. The pipeline looks like this:

  1. Start with a raw text dataset (a corpus).
  2. Apply text preprocessing: tokenization, removing stop words, stemming/lemmatization, and more.
  3. Convert the cleaned text into numbers, since models can't understand raw text directly. Common approaches include TF-IDF, Bag-of-Words, N-grams, and more advanced word-to-vector techniques like Word2Vec and Word Embeddings.
  4. Feed the numeric representation into a model — anything from a classic classifier (e.g. the classic spam vs. ham email classification example) to advanced architectures like RNNs, LSTMs, GRUs, Encoder-Decoder models, and Transformers (e.g. BERT).

This four-part series focuses on the very first stage of that pipeline — cleaning and preparing raw text — starting with Tokenization, then Stop Word Removal, then Stemming, and finally Lemmatization.

Tokenization

What is it?

Tokenization is the process of breaking a paragraph or corpus into smaller pieces called tokens. A token can be a word or a sentence, depending on the type of tokenization used. Several libraries can do this — the two the course focuses on are NLTK and spaCy, with NLTK being the primary tool used in the examples. The two most common functions are:

  • Word Tokenizer — splits a paragraph into individual words, so each word can later be converted into numbers.
  • Sentence Tokenizer — splits a paragraph into individual sentences.

How word tokenization actually works

Take the example sentence:

Input"Let's go to New York!"

The tokenizer processes this in stages:

  1. Splitting on whitespace — the text is first broken wherever there's a space between words.
  2. Prefix handling — anything attached before a word (like a leading punctuation mark) gets separated from it.
  3. Exception handling — the tokenizer recognizes contractions, i.e. two words that have been merged with an apostrophe, like Let's. It intelligently splits this into Let and 's.
  4. Suffix handling — anything attached after a word (trailing punctuation, like the apostrophes around "New York" or an exclamation mark) gets separated too. You can also manually control which punctuation marks should trigger a split.
Smart splitting: The algorithm is smart about compound expressions. For a multi-word name like New York, it recognizes both words belong together as a single unit and keeps them associated — it doesn't just arbitrarily split every space-separated chunk. Similarly, a name like King Kong is understood as one entity rather than two unrelated tokens. For languages that don't use apostrophes the same way (e.g. Chinese, Hindi, Japanese), the tokenizer instead relies on implicit meaning — checking whether two adjacent words form a meaningful unit together before deciding how to split them.

Code walkthrough (NLTK)

First, install NLTK:

pip install nltk

Then download the required NLTK resources/packages, import the tokenizer functions, and run them:

import nltk
nltk.download()  # downloads all NLTK packages/resources

from nltk.tokenize import sent_tokenize, word_tokenize

text = "Hello! How are you today."

# Sentence tokenization — splits on punctuation like ! ? .
print(sent_tokenize(text))

# Word tokenization — splits into individual words
print(word_tokenize(text))
FunctionBehavior
sent_tokenize() Splits the paragraph into separate sentences based on punctuation cues such as periods, exclamation marks, and question marks.
word_tokenize() Splits a sentence into individual words, each returned in its own quotes — even punctuation marks (like ! or .) become separate tokens. It correctly keeps compound words like "New York" together and correctly separates contractions like "you're" into "you" and "'re" — something Python's plain .split() can't do, since it would keep "you're" as one single chunk.

Stop Words

What is it?

Just as Computer Vision techniques (like SIFT and FAST) extract only the important pixels/features from an image so a model can focus on what matters, NLP has an equivalent idea for text. In any paragraph, some words carry very little meaning or importance for the model — words like "the," "and," "a," "of," "was". These low-value, frequently-occurring words are called stop words, and removing them helps the model focus on the words that actually matter.

Where do stop word lists come from?

You can pull a ready-made list of stop words from libraries like NLTK or spaCy, or you can build your own custom stop word list. NLTK's stop word corpus supports many languages — you simply specify the language (e.g. Arabic, English, German) and it returns the full list of stop words for that language.

Code walkthrough

from nltk.corpus import stopwords
from nltk.tokenize import word_tokenize

nltk.download('stopwords')

# Get stop words for a specific language
arabic_stopwords = set(stopwords.words('arabic'))
english_stopwords = set(stopwords.words('english'))
german_stopwords  = set(stopwords.words('german'))

Example: removing stop words from an Arabic business sentence meaning "The company released detailed data about its performance in the first quarter of this year". Words like "in" and "from" are stop words here and get removed:

text = "..."  # original sentence

# 1. Tokenize the sentence into words
words = word_tokenize(text)

# 2. Load the stop word set for the matching language
stop_words = set(stopwords.words('arabic'))

# 3. Filter out any word that is a stop word
filtered_words = [w for w in words if w not in stop_words]

# 4. Rejoin the remaining words into a sentence
result = ' '.join(filtered_words)
print(result)
Same logic, different language Input: "Hello! How are you today." → after removing stop words like "how," "are," "you" → only the meaningful content words remain.

Why bother?

  • Reduces training time, since the model processes less (and less noisy) data.
  • Improves accuracy, because the model concentrates on the words that actually carry meaning.
Use your judgment: If you find your model's accuracy isn't great, you might need to bring back some of the removed words. You're not locked into the default list — you can build your own custom stop word set (as a Python set) tailored to your specific corpus and apply it instead.

Stemming

What is it?

Stemming removes any prefix or suffix attached to a word and reduces it down to its root/stem. For example:

Original wordStem
runningrun
catscat
jumpingjump

Why use it?

Stemming is widely used in applications like text classification and text clustering. Imagine building your own dictionary/corpus by hand: it wouldn't make sense to store every single inflection of a word separately — go, going, gone, goes, and so on. Instead, words are reduced back to their root so the corpus contains fewer unique words. This keeps the dictionary efficient: you can pack in a huge amount of text, but the vocabulary itself stays compact, without repeating conjugations of the same core word.

Code walkthrough (Porter Stemmer)

NLTK offers several stemming algorithms — the three main ones are the Porter Stemmer, the Snowball Stemmer, and the Lancaster Stemmer. The examples use the Porter Stemmer: 

from nltk.stem import PorterStemmer

stemmer = PorterStemmer()
words = ["jumping", "jumps", "jumped", "jumping"]

for word in words:
    print(word, "->", stemmer.stem(word))
    # jumping -> jump
    # jumps   -> jump
    # jumped  -> jump

In practice, you'd first tokenize a sentence into words and then apply the stemmer to each token, printing the original word next to its stem.

Where it's useful

You can use a word's stem/root to search within your corpus or to perform classification — all while not losing the overall meaning of the sentence.

A preview of the next topic: Sometimes the Porter Stemmer (or stemming in general) returns a result that's incorrect or doesn't make real sense as a word. There's a better technique for this — Lemmatization — covered next.
Video 4

Lemmatization

The problem with stemming

Stemming reduces a word to its root, and for many words that's harmless — e.g. reducing several related words down to job doesn't affect meaning. But look at what happens with a word like "information": applying stemming to information, informative, and computers can produce meaningless fragments like "inform," "inform," and "comput." That has no real meaning — if you were building a chatbot and it responded to you with fragments like that, you wouldn't understand it at all. This is the core weakness of stemming: it doesn't always know how to correctly reduce a word to a real root.

How lemmatization is different

Lemmatization is smarter: rather than blindly chopping affixes, it looks at the context of the word — it tries to understand the meaning behind the sentence — and based on that returns the correct base form. For example, lemmatization would correctly return information, informative, and computer/computers as real, valid words, unlike stemming's broken fragments.

Because of this, lemmatization is far more widely used in real NLP applications than stemming. That said, stemming still has its place — it's fine to use when preserving the exact context of a word isn't that important to you. If context matters, lemmatization is the better choice, since it returns words appropriately based on their meaning in the sentence.

Code walkthrough (WordNet Lemmatizer)

from nltk.stem import WordNetLemmatizer
from nltk.tokenize import word_tokenize

lemmatizer = WordNetLemmatizer()

text = "..."  # example sentence
words = word_tokenize(text)

for word in words:
    print(word, "->", lemmatizer.lemmatize(word))

The workflow mirrors stemming: tokenize the sentence into words, then feed each token into the lemmatizer instead of the stemmer, and compare the outputs.

Side-by-side comparison

Takeaway: Lemmatization is generally the better, more reliable choice because it accounts for context and meaning — but stemming remains a valid, lighter-weight option for cases where exact context doesn't matter much.
Wrap-up

Summary Comparison

TechniquePurposeKey NLTK Tool
Tokenization Split raw text into sentences or words so it can eventually be converted into numbers sent_tokenize(), word_tokenize()
Stop Word Removal Discard low-value, high-frequency words to speed up training and sharpen accuracy stopwords.words(language)
Stemming Reduce words to a root form to shrink vocabulary size; fast but sometimes produces non-words PorterStemmer, SnowballStemmer, LancasterStemmer
Lemmatization Reduce words to a valid base form using context; more accurate, more widely used in practice WordNetLemmatizer

Together, these four steps form the foundation of text preprocessing in any NLP pipeline — setting the stage for turning cleaned text into numeric features (via TF-IDF, Bag-of-Words, Word2Vec, etc.) that can be fed into machine learning or deep learning models.
